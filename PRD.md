# PRD: Subway Time 학습팩 (가칭)

## 1. 한 줄 정의

지하철/버스 이동 시간을 기준으로,  
**GeekNews · 프런트엔드 공식 문서 · 업무 연관 자료**를 AI로 큐레이션하여  
“지금 남은 시간 안에 끝낼 수 있는” 학습팩을 자동으로 추천해 주는 개인용 학습 도구입니다.

---

## 2. 배경 및 문제 정의

### 2.1 배경

- 인턴십, 학교, 외주 작업을 병행하면서 **집중해서 공부할 수 있는 긴 시간**을 확보하기가 어렵습니다.
- 남는 시간은 대부분 **이동 시간(지하철/버스)**로 존재하지만,
  - “지금 이 15분 동안 뭘 보면 좋지?”  
  - “내가 요즘 하고 있는 일과 연결되는 학습거리가 뭐였지?”  
  와 같은 고민 때문에 결국 SNS/유튜브로 흘러가게 됩니다.
- 특히 프런트엔드 개발은
  - 매일 새로운 기술과 레퍼런스가 쏟아지고,
  - 공식 문서/블로그/레딧/GeekNews 등 **정보의 바다** 속에서
  - “지금 나에게 꼭 필요한 것만 골라서” 공부하는 것이 매우 어렵습니다.

### 2.2 문제 정의

1. **이동 시간은 분명 있지만, 무엇을 공부할지 고르다가 시간을 날리는 문제**
2. **최신 프런트엔드 정보는 많은데, 내 현재 업무/실력과 연결된 학습 경로가 없는 문제**
3. **‘읽기만 하고 끝나는’ 공부가 되어, 실제 업무와 연결되지 않는 문제**
4. **오늘 뭘 공부했는지, 반복해서 볼 가치가 있는지 기록/회고가 잘 안 되는 문제**

---

## 3. 목표 및 성공 지표

### 3.1 제품 목표

- 이동 시간을 **즉시 실행 가능한 분 단위 학습 세션**으로 바꿔주는 개인 학습 도구를 만든다.
- GeekNews, 공식 문서, 최근 업무 로그를 AI로 요약·조합하여  
  **“지금의 나에게 의미 있는 10~30분짜리 학습팩”**을 생성한다.
- 4주간 실사용을 통해,
  - 실제 업무나 사이드 프로젝트에 도움이 된 사례를 최소 2건 이상 만든다.

### 3.2 성공 지표 (1차 MVP 기준)

정량 지표(개인 사용 기준):

- 4주 동안 **학습팩 실행 횟수 20회 이상**
- 한 번 실행 시 **실제 읽거나 활용한 콘텐츠 2개 이상**
- 동일 주제/기술에 대한 **반복 노출(리마인드) 3회 이상** 발생

정성 지표:

- “지하철 타면 그냥 이 앱 켠다”는 **자연스러운 루틴 형성 체감**
- “이걸 보고 바로 코드/문서에 적용했다”는 **업무 연결 사례 2건 이상**

---

## 4. 대상 사용자 & 페르소나

### 4.1 1차 타깃

- 20대 중후반, **프런트엔드 개발자/인턴/학생**
- 학교/회사/외주를 병행하며 **시간은 쪼개져 있지만 의지는 있는 사람**
- 영어 공식 문서/블로그도 어느 정도 읽을 수 있지만, **어디서부터 봐야 할지 막막한 사람**

### 4.2 대표 페르소나

**민(26) – 프런트엔드 인턴 + 대학생**

- 평일 기준 출퇴근/이동 시간으로 하루 1~2시간 정도가 있음.
- “오늘은 리액트 쪽을 공부해야지” 생각하지만, 막상 지하철 안에서는 SNS/유튜브를 켠다.
- GeekNews, 트위터, 블로그를 팔로우해두었지만 **북마크만 쌓이고** 제대로 읽는 경우는 적다.
- 최근 회사에서 다뤘던 기술(React, TypeScript, 디자인 시스템 등)과 연결된 공부를 하고 싶어 함.

---

## 5. 핵심 컨셉

### 5.1 지하철 타임에 꼭 들어갈 4가지

1. **GeekNews 일일 알림**  
   - 최신 프런트엔드 관련 이슈/글 중, 오늘 읽을만한 것 1~2개 추천
2. **프런트엔드 공식 문서 요약 (Gemini API 기반)**  
   - React, Next.js, TypeScript 등 공식 문서를 “지금 남은 시간에 읽을 수 있는 분량”으로 요약
3. **나의 최근 작업과 연결된 자료 (Agent + Notion 업무일지)**  
   - Notion 업무 일지에서 키워드를 읽어, 관련 문서/레퍼런스/예제를 추천
4. **지도 API + 시간 조절**  
   - 출발·도착 기준으로 오늘 남은 이동 시간을 계산하고  
     → 그 시간 안에 끝낼 수 있도록 학습 콘텐츠를 조합·분량 조절

### 5.2 핵심 가치

- “뭘 공부할지 고민하는 시간” 없이, **앱을 켜면 바로 오늘의 학습팩이 나온다.**
- “내가 요즘 하고 있는 일”과 연결된 콘텐츠 위주로 추천한다.
- “이동 시간 = 공부 시간”이라는 **새로운 루틴을 만들어주는 도구**가 된다.

## 6. 주요 기능 정의

### 6.1 홈 – 오늘의 지하철 타임

**설명**  
- 앱을 열면 바로 오늘 기준 기본 학습팩이 보이는 화면입니다.

**사용자 스토리**

- “사용자로서, 아침에 지하철을 탈 때 앱을 열면,  
  오늘 남은 이동 시간에 맞는 ‘오늘의 학습팩’을 바로 보고 싶습니다.”

**주요 요소**

- 오늘 날짜 / 요일 / 기본 이동 시간(사용자가 설정한 값)
- 오늘의 학습팩 카드 리스트 (예: 3~5개)
  - 각 카드: 제목, 소요 시간(예: 7분), 출처 아이콘(GeekNews/Docs/Notion 등)

---

### 6.2 이동 시간 기반 학습팩 생성

**설명**  
- 사용자가 ‘오늘 이동 시간’을 입력(또는 지도 기반 자동 계산)하면, 그 시간 안에서 소화 가능한 학습팩을 자동 구성합니다.

**사용자 스토리**

- “사용자로서, 오늘 출퇴근 시간이 25분이라면,  
  25분 안에 끝낼 수 있는 콘텐츠 묶음을 자동으로 받고 싶습니다.”

**기능 요구사항**

- 사용자는 다음 중 하나로 시간을 설정할 수 있습니다.
  - 직접 입력: 10, 20, 30, 40분 등
  - 지도 기반: 출발역/도착역 선택 → 예상 소요 시간 가져오기(1차 버전은 단순 고정값도 허용)
- 시스템은 설정 시간 안에 들어가는 카드들을 우선순위에 따라 조합합니다.
  - 1순위: Notion 업무 연관 카드
  - 2순위: GeekNews 카드
  - 3순위: 공식 문서 요약 카드

---

### 6.3 GeekNews 연동 카드

**설명**  
- GeekNews의 프런트엔드 관련 글을 가져와 “오늘 읽을만한 글 카드”로 변환합니다.

**기능 요구사항**

- 최소 기능:
  - GeekNews RSS 또는 스크랩된 URL 목록을 수동/반자동으로 관리
  - 각 URL을 Gemini로 요약 (5분 내로 읽을 분량)
- 카드 정보:
  - 제목, 한 줄 요약, 예상 소요 시간, 링크 열기 버튼

---

### 6.4 공식 문서(Google/React/Next 등) 요약 카드

**설명**  
- React, Next.js, TypeScript 등 공식 문서의 일부를 요약하여 카드로 제공합니다.

**기능 요구사항**

- 특정 문서 URL/섹션을 사전에 설정해두고,  
  Gemini API를 통해 “요약 + 꼭 알아야 할 포인트 3개” 형식으로 변환
- 각 카드당
  - 제목(문서 섹션 제목)
  - 요약 본문 (5~10문장)
  - “오늘은 여기까지 읽기” 표시
- 추후, 사용자가 이미 읽은 섹션은 우선순위에서 제외하는 로직 고려

---

### 6.5 Notion 업무 일지 연동 + Agent 추천

**설명**  
- Notion에 기록된 최근 업무 일지를 읽고, 키워드를 추출하여 관련 공식 문서/자료를 추천합니다.

**기능 요구사항 (1차 버전)**

- Notion에서 최근 N일(예: 7일) 이내 업무 로그를 가져온다. (초기에는 수동 export도 허용)
- 로그 텍스트에서 주요 키워드 추출 (예: “React Query”, “i18n”, “Electron” 등)
- 추출된 키워드를 기반으로 **사전에 매핑해둔 자료 목록**에서 추천
  - (예: “React Query” → 공식 문서 링크 + 블로그 1개)
- 추천 결과를 카드로 생성
  - “최근에 이런 일을 했으니, 이런 문서를 보면 좋겠습니다.” 형식으로 안내

---

### 6.6 지도 API 연동 (이동 시간 계산)

**설명**  
- 출발역/도착역을 기반으로 오늘 이동 시간을 계산하거나,  
  최소한 사용자가 입력한 이동 시간을 시각적으로 보여줍니다.

**기능 요구사항 (MVP)**

- 1차: 시간 직접 입력 + 단순 지하철 노선도/지도 UI만 제공
- 2차: 카카오/네이버/Google 지도 API 연동, 소요 시간 조회
- 조회된 시간(분)을 학습팩 생성 로직에 넘겨서 카드 조합

---

### 6.7 학습 기록 & 회고

**설명**  
- 오늘 어떤 학습팩을 실행했는지, 무엇을 실제로 읽었는지 기록합니다.

**기능 요구사항**

- 학습팩 실행 시:
  - 시작/종료 시간 기록
  - 사용자가 “완료” 체크한 카드 수 기록
- 요약 리포트:
  - 이번 주 총 학습 시간
  - 가장 많이 본 주제/키워드
  - 다시 보면 좋을 카드(즐겨찾기) 목록

---

## 7. 사용자 플로우 & 화면 구성

### 7.1 핵심 플로우 (텍스트 다이어그램)

1. 앱 실행  
2. (처음인 경우) 기본 이동 시간 설정 (예: 출근 30분, 퇴근 20분)
3. 홈에서 오늘의 학습팩 제안 확인
4. 출발 직전에 “오늘 이동 시간 조정” (ex. 25분)
5. 학습팩 카드 목록 확인 → 순서대로 읽기
6. 도착 후 “완료/미완료” 체크
7. 기록 및 리포트에서 이번 주 학습 현황 확인

### 7.2 예상 화면 목록

1. 온보딩 / 기본 설정 화면
2. 홈 – 오늘의 학습팩
3. 이동 시간 설정(직접 입력 + 지도)
4. 학습팩 상세 화면(카드 리스트)
5. 개별 카드 상세(GeekNews / Docs / Notion 추천)
6. 학습 기록 및 리포트 화면

## 8. 기술 스택 (초안)

### 8.1 프론트엔드

- Netx.js (PWA)
- TypeScript
- UI: TailwindCSS 또는 간단한 컴포넌트 라이브러리

### 8.2 백엔드 / API

- 1차: 별도 서버 없이,  
  - 정적 데이터(문서 URL, 매핑 테이블)를 JSON으로 관리
  - 클라이언트에서 AI API 직접 호출 (토큰/보안 고려 필요)
- 2차 이후: 간단한 BFF(Backend For Frontend) 또는 Edge Function 검토

### 8.3 AI

- Gemini API  
  - 입력: URL 또는 본문 텍스트  
  - 출력: 요약, 핵심 포인트, 예상 소요 시간 등

### 8.4 외부 서비스

- 지도 API: 카카오/네이버/Google 중 선택
- Notion API: 업무 일지 읽기 (초기에는 수동 export로 대체 가능)
- GeekNews: RSS 또는 Scraping 결과 활용

## 9. 범위 정의 (MVP 스코프)

### 9.1 이번 AIM 버전에서 반드시 포함

- 홈 화면 + 오늘의 학습팩 보기
- 이동 시간 직접 입력 + 학습팩 조합 로직
- GeekNews / 공식 문서 / Notion 기반 카드 최소 1~2종씩 구현
- 학습 완료 체크 + 간단한 기록 보기

### 9.2 이번 버전에서 제외 (향후 고려)

- 완전 자동의 Notion 연동 (OAuth 등 포함)
- 고도화된 추천 알고리즘 (가중치 학습, 사용자 행동 기반 자동 튜닝)
- 멀티 사용자/서비스화(다른 사람들도 쓸 수 있는 수준의 계정 시스템)
- 정교한 지하철 실시간 도착 정보 반영

## 11. 리스크 및 대응

1. **AI 비용 및 속도**
   - 리스크: 자주 호출하면 비용/속도 문제가 생길 수 있음  
   - 대응: 1차 버전에서는 소수의 문서/URL에 한정해 요약, 캐싱 전략 도입

2. **시간 관리(본업 병행)**
   - 리스크: 인턴/학교/외주로 인해 개발 시간이 부족  
   - 대응: 각 주차에 “최소 기능(MVP 단위)” 명확히 정의, 기능보다 흐름 우선

3. **지도/Notion 등 외부 API 복잡도**
   - 리스크: 인증/쿼터 등으로 인해 구현이 지연될 수 있음
